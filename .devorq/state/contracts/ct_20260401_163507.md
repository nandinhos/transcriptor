# CONTRATO DE ESCOPO - Expandir transcritor com suporte a grandes arquivos, múltiplos uploads, fila de execução, embeddings e chat de estudos

## IDENTIFICAÇÃO
- **Task**: Expandir transcritor com suporte a grandes arquivos, múltiplos uploads, fila de execução, embeddings e chat de estudos
- **Tipo**: feature
- **Complexidade**: alta
- **Estimativa**: 4-6 horas

## FAZER
1. Suporte a arquivos de vídeo/áudio acima de 500MB (streaming/chunking)
2. Verificador de modelo Whisper já baixado em disco (cache local)
3. Sistema de múltiplos arquivos simultâneos
4. Fila de execução com status (pendente, processando, concluído, erro)
5. Salvar transcrições em arquivos .txt
6. Sistema de validação de transcrição (usuário confirma texto)
7. Geração de embeddings das transcrições validadas (sentence-transformers)
8. Chat integrado em aba separada para estudos sobre conteúdo vetorizado

## NÃO FAZER
1. Não implementar transcrição em tempo real (streaming de áudio)
2. Não integrar com banco de dados externo (usar JSON local)
3. Não implementar autenticação de usuários
4. Não usar modelos de embedding pesados (usar modelos leves como all-MiniLM-L6-v2)

## ARQUIVOS PERMITIDOS
- app.py (Streamlit principal)
- requirements.txt
- lib/model_manager.py (gerenciamento de modelos)
- lib/queue_manager.py (fila de processamento)
- lib/embeddings.py (geração de embeddings)
- lib/storage.py (armazenamento local)
- chats/ (diretório para histórico de chats)
- transcripts/ (diretório para transcrições salvas)

## ARQUIVOS PROIBIDOS
- Não modificar arquivos do .devorq/
- Não modificar bin/

## DONE_CRITERIA (Objetivos)
- [ ] Arquivos até 1GB podem ser enviados sem erro
- [ ] Modelo Whisper é verificado antes do download
- [ ] Múltiplos arquivos podem ser adicionados à fila
- [ ] Status em tempo real da fila de processamento
- [ ] Transcrições são salvas em .txt
- [ ] Usuário pode validar/editrar transcrição antes de vetorizar
- [ ] Embeddings são gerados após validação
- [ ] Chat na aba separada consulta conteúdo vetorizado

## RISCO_IDENTIFICADO
- Memória insuficiente para arquivos muito grandes
- Tempo de processamento longo para vídeos grandes
- Embeddings podem consumir muita memória RAM
